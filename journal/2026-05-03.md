## 判断・設計メモ

### Phase 2.5の目的

- 実ログを何度も開き、schemaを確認し、channelを選び、load/addして波形確認する一連の操作を速くする
- 解析機能を増やす前に、実地検証の回転数を上げる
- 現状の課題は以下:
  - Parquet pathが直打ち
  - channel選択とload操作が離れている
  - loaded / loading / unloaded状態が一覧で分かりにくい

### 作図寄り機能の扱い

- 正式な `.pyc.json` 互換や作図設定仕様はPhase 4へ残す
- Phase 2.5では、Rust版内部の軽量な `session preset` として扱う
- 保存対象は、実地検証で便利な最小範囲に限定する:
  - parquet path
  - row構成
  - channel path
  - Line / Step
  - visible
  - color override
  - line width
  - Y Auto / Manual range
  - X range

### 複数channel選択の優先度

- channel一覧で複数チェックして `Load checked` するUIは、現時点では優先度を下げる
- 理由:
  - 実地検証では、まとめて選ぶことより目的channelへ速く到達することの方が効きそう
  - 同時ロード時にtime列の重複読み込み回避、部分失敗表示、同一Row内の重複扱いを整理する必要がある
  - 100ch対応としては、検索、状態フィルタ、sort、selected detailの方が低リスクで効果が大きい
