# Rust移植 Phase 0 UI操作仕様

---

## 1. 対象

本仕様は `code/proto_3_1b/app.py` のUI動作をRust版移植の参照として記録する。
Rust版では実装方式を変えてよいが、ここに記録したユーザー操作と期待動作は原則として維持する。

---

## 2. 画面構成

### ヘッダーバー

左から以下を配置する。

- アプリ名: `PanelY`
- Parquetファイルパス入力
- `読み込み` ボタン
- 作図設定パス入力
- 作図設定保存ボタン
- 作図設定ロードボタン
- 作図設定ステータス
- スクロールズーム固定切替
- 計測モード切替
- データ読み込みステータス
- テーマ切替
- ヘッダー固定切替
- クレジット表示

### 行定義エリア

- `+ 行追加`
- `更新 (Ctrl+Enter)`
- `行設定` 折りたたみ
- 行ごとのチャンネル選択
- 行ごとのstep対象選択
- 行ごとのY軸min/max指定
- 行削除ボタン

### チャンネル設定パネル

- `チャンネル設定（色・太さ）` 折りたたみ
- 使用中チャンネルごとの色選択
- 使用中チャンネルごとの線幅入力

### 解析パネル

- Cursor A/B と Δt 表示
- 区間統計表
- FFTコントロール
- FFTグラフ

### 波形表示エリア

- ヘッダー行: `Channel / Value`, `Waveform`
- 行ごとのラベル列
- 行ごとの波形グラフ
- 最終行のみX軸ラベルを表示

## 3. データ読み込み操作

### Parquetパス入力

操作:

1. ファイルパス欄にパスを入力する
2. 入力中に候補が表示される
3. 候補をクリックすると入力欄に反映される

期待動作:

- ディレクトリ候補は末尾 `/` 付きで表示される
- `.parquet` ファイルだけが読み込み候補になる
- 候補は最大30件

### 読み込み

操作:

1. `.parquet` ファイルを指定する
2. `読み込み` を押す

期待動作:

- ファイルが存在しない場合はエラー表示
- ディレクトリ指定の場合はエラー表示
- `.parquet` 以外はエラー表示
- `time` 以外の列をチャンネルとして認識
- チャンネルが0本ならエラー表示
- time列が非単調ならソートし、警告をステータスに付ける
- 初期行はチャンネルごとに最大8行まで自動生成
- 作図設定パス欄に `{データファイル名}.pyc.json` が入る

---

## 4. 行設定操作

### 行追加

操作:

- `+ 行追加` を押す

期待動作:

- 末尾に空の行設定が追加される
- 行番号は追加時点の `row-count` を使う

### 行削除

操作:

- 対象行の削除ボタンを押す

期待動作:

- 対象行だけ削除される
- 最後の1行は削除しても空状態にしない
- パターンマッチcallbackの誤発火では削除しない

### チャンネル選択

操作:

- 行ごとのドロップダウンで1つ以上のチャンネルを選ぶ

期待動作:

- 同じ行に複数チャンネルを重ね表示できる
- 空行は波形生成時に無視される

### step表示

操作:

- 行ごとのstepドロップダウンで対象チャンネルを選ぶ

期待動作:

- 対象チャンネルは階段状に表示される
- 現行Dash版では Plotly の `line_shape="hv"` 相当

### Y軸範囲指定

操作:

- 行ごとの `Y: min ~ max` に数値を入力する

期待動作:

- min/max両方が指定された場合はY軸範囲を固定
- minのみ、maxのみの場合は片側範囲として扱う
- 空欄の場合は自動範囲

---

## 5. 波形更新操作

操作:

- `更新 (Ctrl+Enter)` を押す
- または `Ctrl+Enter` / `Cmd+Enter` を押す

期待動作:

- 行設定から波形行を生成する
- 空でない行だけを表示対象にする
- 使用中チャンネルだけでチャンネル設定パネルを再生成する
- 既存の色・線幅設定は維持する
- 現在のX軸ズーム範囲があれば維持する

---

## 6. 波形操作

### ホバー

操作:

- 任意の波形上にマウスを置く

期待動作:

- ホバーX座標を全行に同期する
- 全行に縦スパイクラインを表示する
- 各行左側に、最も近い時刻のチャンネル値を表示する

### ズーム・パン

操作:

- Plotly標準操作でX軸をズームまたはパンする

期待動作:

- X軸範囲を全行に同期する
- `xaxis.range[0]` / `xaxis.range[1]` 相当を共有する
- autorange操作では全体表示に戻す
- ズーム範囲に応じて描画データを再抽出する

### スクロールズーム固定

初期状態:

- 表示ラベルは `スケール固定`
- `scrollZoom` はFalse

操作:

- ボタンを押す

期待動作:

- 奇数クリックで `scrollZoom=True`
- 偶数クリックで `scrollZoom=False`
- ラベルとスタイルが切り替わる

---

## 7. 計測操作

### 計測モード

初期状態:

- OFF

操作:

- `計測` ボタンを押す

期待動作:

- 奇数クリックでON
- 偶数クリックでOFF
- OFFにするとCursor A/Bをクリアする

### Cursor A/B配置

操作:

1. 計測モードONにする
2. 波形をクリックする
3. もう一度波形をクリックする
4. さらにクリックする

期待動作:

- 1回目クリックでCursor Aを配置
- 2回目クリックでCursor Bを配置
- Bがある状態で次にクリックすると、新しいCursor Aを配置し、Bはクリア
- Aは青系、Bは赤系の破線で表示

### 区間統計

期待動作:

- A/B両方がある場合に統計を表示
- 対象は現在表示対象になっている全チャンネル
- 重複チャンネルは1回だけ表示
- 表示項目は `A`, `B`, `Delta`, `Mean`, `Max`, `Min`, `P-P`, `RMS`, `Slope/s`

---

## 8. FFT操作

表示条件:

- Cursor A/Bが両方配置されている

操作:

1. FFT対象チャンネルを選ぶ
2. 窓関数を選ぶ
3. Y軸表示を選ぶ
4. 周波数範囲を必要に応じて入力する
5. `FFT` を押す

期待動作:

- 対象チャンネルは最大3件
- 窓関数は Hanning / Hamming / Rectangular
- Y軸は Amplitude / dB
- 周波数範囲は min/max を指定可能
- FFT対象区間はCursor A/B間
- 信号から平均値を引いてDC除去する
- FFTパネルを表示する
- Cursor変更時はFFTパネルを隠す

---

## 9. 作図設定操作

### 保存

操作:

1. 作図設定パスを指定する
2. 保存ボタンを押す

期待動作:

- 親ディレクトリがなければ作成する
- UTF-8 JSONとして保存する
- 保存成功時にファイル名を表示する
- 保存先未指定ならエラー表示

### ロード

操作:

1. `.pyc.json` パスを指定する
2. ロードボタンを押す

期待動作:

- JSONを読み込む
- `data_path` のParquetを読み込む
- 行構成、step設定、Y軸範囲、色、線幅、テーマ、scrollZoomを復元する
- Cursor A/Bはクリアする
- 波形を直接復元描画する

既知リスク:

- 作図ロード後にチャンネル分散表示が発生する経路が過去に報告されている。
  Rust版では、ロード時のUI状態更新と波形描画を分離して同種の競合を避ける。
