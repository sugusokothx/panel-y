## 3. 区間統計リファレンス

固定カーソル:

- Cursor A: `0.1 s`
- Cursor B: `0.3 s`

現行Dash版の計算仕様:

- `idx_a = (df["time"] - cursor_a).abs().idxmin()`
- `idx_b = (df["time"] - cursor_b).abs().idxmin()`
- `i_lo, i_hi = min(idx_a, idx_b), max(idx_a, idx_b) + 1`
- 区間統計は両端を含む
- `Slope = (B - A) / (cursor_b - cursor_a)`

| idx_b | 30000 |
| time_a | 0.1 |

## 4. FFTリファレンス

固定カーソル:

- Cursor A: `0.1 s`
- Cursor B: `0.9 s`

FFT条件:

- 対象区間: A/B両端を含む
- 窓関数: Hanning
- DC除去: あり。`seg - seg.mean()`
- 振幅計算: `abs(rfft(windowed)) * 2.0 / N`
- dB計算: `20 * log10(max(amplitude, 1e-12))`

参照情報:

| 項目 | 値 |
|------|----|
| idx_a | 10000 |
| idx_b | 90000 |
| N | 80001 |
| dt | 9.999999999996123e-06 |

ピーク参照値:

|----|----------------|----------|---------|
| sine_50Hz | 49.99937500783179 | 4.9999366897603945 | 13.979290104873243 |
| pwm_1kHz | 999.9875001566357 | 1.2867577545787694 | 2.1899358836646003 |
| chirp_1_500Hz | 249.99687503915894 | 0.2797771990412082 | -11.063753640634989 |

許容誤差:

- peak_freq: `0.1 Hz`
- peak_amp: `1e-4`
- peak_dB: `1e-3 dB`

## 5. Rust版で合わせるべき仕様

- カーソル時刻から最近傍サンプルを選ぶ
- 区間は両端を含める
- 統計は生データで計算する
- FFTも生データで計算する
- 表示用LODやmin/max envelopeを解析に使わない
- FFT前に区間平均を引く
- Hanning/Hamming/Rectangularの定義をNumPy相当に合わせる
