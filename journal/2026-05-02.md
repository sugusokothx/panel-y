## 判断・設計メモ

- previewは「操作中だけ品質を落として体感を改善する」手段だが、イベント探索型の波形ビューアでは目印の見え方が変わるリスクが大きい
- 多少のカクつきより、spike / edge / step / outlierが消える、または操作中と停止後で見え方が変わることの方が問題になりやすい
- 単純間引きは短いspikeやedgeを落とす可能性があるため採らない
- min/max envelopeは描画点数を減らしてもspikeや外れ値の存在を残せるため、Line表示の基本方式として妥当
- Step/digital系はmin/maxだけではedge位置が曖昧になるため、change-point / edge保持を別扱いで維持する
- previewを残す場合も、主戦略ではなくdebug / emergency toggle扱いにする
- `8,000,000 samples` のpreview閾値はメモリ計測から算出した値ではなく、medium 2Mを巻き込まずlarge 10Mだけを対象にする暫定ヒューリスティック
  - 長期的には固定datasetサイズではなく、直近の通常描画時間やvisible sample workに基づくadaptive gateの方が自然

## 詰まったこと・解決策

- pan/zoom中previewを入れたところ、sample数が小さいケースでもpreviewが発動して操作品質を落としうることが分かった
  - dataset sample数、visible sample数、channel数から推定負荷を見て、preview発動条件を絞った
  - さらに、計画その2ではpreview自体を主戦略から外す方針に変更
- Line envelopeはbucket数を減らしても可視sampleの走査コストが残る
  - bucket数を落とすpreviewではなく、overscan cache、bucket境界共有、lazy tile / LODで走査量そのものを減らす方針にした

### overscan range cache

- `ChannelStore` の envelope cache を、表示range完全一致ではなく overscan付きcache rangeで持つように変更

### bucket境界共有 / 一括生成

- `parquet_waveform::MinMaxEnvelopePlan` を追加
  - time rangeから bucket境界を一度だけ生成
  - 各channelは同じ `EnvelopeBucketSpan` を使って min/max を取る
- `EnvelopeBucket` に `start_index` / `end_index` を持たせた
  - overscan cacheから表示rangeへclipするとき、境界bucketだけraw値で再計算できるようにした
- `clip_min_max_envelope_to_range()` を追加
  - overscan範囲外のbucketを描画から除外
  - 表示rangeの境界にまたがるbucketはraw値で再計算
  - Y auto rangeがoverscan範囲の外れ値に引っ張られないようにした

### Line lazy tile / LOD

- `parquet_waveform::LineTileCache` を追加
  - channelごとに固定sample幅 `256 samples/tile` の min/max tileをlazy build
  - tileは描画用cacheであり、raw dataとは分離
- 広域Line表示だけ tile LOD を使うようにした
  - `source_sample_count >= 500,000`
  - `bucket_size >= 512`
- Step fallbackのenvelopeにはLine tileを使わないよう分離
  - Step edge / dense LODは次段階で別途扱う
- tile由来のbucket生成も min/max を保持するため、spike / outlierの存在を落とさない
- GUI cache表示とframe timingに line tile cache の情報を追加
  - tile hits / builds
  - tile memory
  - tile build time

## 追加作業: Step edge cache / dense LOD

### 実装

- `ChannelStore` にStep channel用のedge cacheを追加
  - channelごとに `index/time/value` のchange-point listをlazy build
  - channel reload時に該当cacheを破棄
  - dataset reset時にcacheを破棄
- Step表示経路を更新
  - raw sample Step: visible sample数が `MAX_EXACT_STEP_SAMPLES` 以下
  - cached edge Step: raw sampleが多いがedge数が `MAX_STEP_CHANGE_POINTS` 以下
  - dense Step envelope: edge数が多すぎる場合
- cached edge Stepでは、表示rangeをedge listへbinary searchし、左端直前の状態とrange内edgeだけを抽出する
- dense Step envelopeは `VisibleTraceData::DenseStepEnvelope` としてLine envelopeと区別した
  - 描画はmin/max ribbon
  - Step fallbackではLine tile LODを使わない
- GUI / bench表示を追加
  - frame timingに edge cache hit/build を追加
  - cache表示にStep edge cache数、edge数、memory、build時間を追加
  - bench出力にdense step数、edge hit/build、edge count、edge memoryを追加

### テスト

  - edge listからrange抽出して正確なchange-point Stepを作る
  - 表示range左端で直前状態を引き継ぐ
  - edge数上限超過時にdense fallbackを返す
  - `ChannelStore` がStep edge cacheを再利用する
