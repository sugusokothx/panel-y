# Rust移植 Phase 0 機能チェックリスト

---

## 1. 前提

Rust版の仕様リファレンスは `code/proto_3_1b` とする。

`proto_3_1` には multiconverter exe化対応があるが、ビューア本体の仕様固定では
大容量対応と作図設定統合済みの `proto_3_1b` を正とする。

---

## 2. 優先度定義

| 優先度 | 意味 |
|--------|------|
| Phase 1 | Rust縦切りプロトタイプで必須 |
| Phase 2 | 基本ビューアとして必須 |
| Phase 3 | 解析機能として必須 |
| Phase 4 | 運用・保存・配布として必須 |
| 後回し | Rust初期版では再現しない |

## 11. Phase 1に持ち込む最小機能

- Parquet読み込み
- time列・チャンネル列認識
- 1ch選択
- 1行波形表示
- pan / zoom
- 表示範囲に応じたmin/max envelope
- 大容量データでフリーズしないことの確認

Phase 1では、Cursor、FFT、作図設定、複数行UIは実装しない。
