## Phase 2継続: P2-06b 高密度stepのエッジ保持

### 確認

- `gate_pwm` は1 MHz sample / 20 kHz PWMのため、中域ズームでもvisible sample数が `MAX_EXACT_STEP_SAMPLES = 12,000` を超えやすい
- 一方で、0.1s-0.3s程度の表示なら変化点数は4,000-12,000程度で、直接Step描画できる範囲だった
- sample数だけでmin/max envelopeへfallbackすると、エッジ時刻を不要に失うため、変化点保持方式は必要と判断
- 確認値:
  - `gate_pwm`
    - 0.1s: 100,001 samples / 4,000 changes -> edge-step対象
    - 0.2s: 200,001 samples / 8,000 changes -> edge-step対象
    - 0.3s: 300,001 samples / 12,000 changes -> edge-step対象
    - 0.5s: 500,001 samples / 20,000 changes -> envelope fallback
  - `pwm_10kHz`
    - 0.2s: 20,001 samples / 4,000 changes -> edge-step対象
    - 0.5s: 50,001 samples / 10,000 changes -> edge-step対象
    - 1.0s: 100,001 samples / 20,000 changes -> envelope fallback

### 実装

- `MAX_STEP_CHANGE_POINTS = 12,000` を追加
- Step表示の選択順を以下に変更
  - raw sample Step: visible sample数が `MAX_EXACT_STEP_SAMPLES` 以下
  - change-point Step: visible sample数は多いが変化点数が `MAX_STEP_CHANGE_POINTS` 以下
  - min/max envelope: 変化点数も多すぎる場合
- change-point Stepでは、表示範囲開始直前の状態を引き継ぎ、値が変わったsample時刻だけを保持する
- GUI status / plot labelにedge-step数を表示するようにした

### 検証

- `cargo fmt`

- `cargo test`
  - 16 tests passed
  - 追加テスト:
    - sample数が多い場合でも変化点だけを保持する
    - 表示範囲開始直前の状態をchange-point Stepでも引き継ぐ
    - 変化点数上限を超えた場合はchange-point Stepを避ける
